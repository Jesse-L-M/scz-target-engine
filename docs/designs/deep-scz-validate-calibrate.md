## Vision

The 10x version of this project is not "more features." It is the first open-source
target prioritization system that can show it would have changed historical decisions
correctly.

### 10x Check

- Historical backtesting against real program outcomes (30-50 SCZ targets, Phase 2+)
- Calibrated confidence intervals replacing binary stability labels
- Living evidence pipeline with automated refresh, drift detection, change attribution
- Prospective forecast scorecards with public self-accountability
- Rank-change attribution showing exactly what moved and why
- "Why not this target?" query answering the most common researcher question

### Platonic Ideal

A researcher opens the observatory and sees a living evidence landscape for schizophrenia
targets. Each target has a provenance-rich card showing exactly where the evidence comes
from, how fragile the ranking is, what failures preceded it, and what would falsify it.

The system maintains a public ledger of forecasts it made and how they turned out.
Visitors can see: "In 2026, the engine predicted Target X would fail translation.
Here is the pre-registered forecast, the evidence at that time, and the outcome."

When new evidence arrives, the system automatically re-scores, flags what changed,
and alerts on targets whose rankings shifted significantly.

## Architecture Notes

The existing additive layering philosophy (v0 -> v1 -> v2 -> policy -> packets) is
architecturally sound and supports all accepted expansions without mutation.

Key architectural decisions for implementation:

1. Historical backtesting should follow the rescue governance pattern: frozen evidence
   snapshots with checksum validation and explicit leakage boundaries.
2. Bootstrap CIs should extend `run_stability_analysis()` rather than creating a
   parallel path.
3. The refresh pipeline wraps existing source adapters in an orchestration layer with
   retry, drift detection, and structured failure reporting.
4. Rank attribution consumes two complete build outputs and decomposes the per-entity
   delta into per-layer contributions.

## Eng Review Adjustments (2026-03-31)

The eng review and Codex outside voice produced 4 scope adjustments:

1. **Historical backtesting resequenced**: Data curation (30-50 SCZ programs) is now an
   explicit parallel workstream alongside engineering. The CT.gov adapter and replay
   infrastructure are Phase 2 engineering; the corpus curation is parallel research.
2. **Refresh pipeline demoted**: Deferred to TODOS.md. The gene scoring path is still
   seed-driven, so automated refresh would mostly re-fetch curated shortlist inputs.
   Blocked by full-universe gene scoring.
3. **Bootstrap CIs are additive**: New output fields alongside existing `heuristic_stable`,
   not a replacement. No v0 contract change.
4. **"Why not?" split implementation**: Targets with hypothesis packets use existing packet
   fields (decision_focus, evidence_anchors, risk_digest). Targets without packets get a
   lightweight bottleneck decomposition from scoring data.
